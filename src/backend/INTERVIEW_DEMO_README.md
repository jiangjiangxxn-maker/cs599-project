# 面试模拟演示使用说明

## 快速开始

### 运行演示脚本

```bash
cd backend
python demo_interview.py
```

### 演示内容

该脚本模拟了一个完整的后端技术面试流程，包括：

1. **开场**：面试官介绍面试流程
2. **多轮问答**：5轮技术问题提问与回答
3. **评分反馈**：每轮回答后给出评分和反馈
4. **最终评估**：面试结束后的综合评估报告

## 面试流程示例

```

======================================================================

场景：应届生后端开发岗位技术面试
面试官：AI 面试官
候选人：模拟用户
```

### 第1轮：开场

**候选人**：模拟后端技术面试

**面试官**：你好，欢迎参加今天的后端开发校招技术面试...

**问题**：首先我们聊聊计算机网络基础。请详细描述一下TCP三次握手的过程...

### 第2-5轮：问答与评分

每轮包含：
- 候选人回答
- 面试官反馈（评分 + 改进建议）
- 下一个问题或结束面试

## 自定义面试流程

### 修改用户输入序列

编辑 `demo_interview.py` 文件中的 `user_inputs` 列表：

```python
user_inputs = [
    "模拟后端技术面试",  # 初始请求
    "你的回答内容...",    # Q1 回答
    "你的回答内容...",    # Q2 回答
    "你的回答内容...",    # Q3 回答
    "你的回答内容...",    # Q4 回答
    "结束面试",           # 结束请求
]
```

### 面试模式选择

在初始请求中指定面试类型：

```python
# 技术面试（默认）
"模拟后端技术面试"

# HR面试
"模拟HR面试"

# 项目深挖
"模拟项目深挖面试"
```

## 技术实现

### 核心组件

1. **InterviewerAgent**：面试官智能体
   - Phase A：开场 + 第一个问题
   - Phase B：多轮对话 + 评分
   - Phase C：最终评估报告

2. **Orchestrator**：协调器
   - 意图识别
   - Agent 路由
   - 状态管理

3. **Memory Manager**：会话管理
   - 对话历史保存
   - 状态持久化

### 关键特性

- ✅ **智能评分**：每轮回答后自动评分（0-10分）
- ✅ **实时反馈**：指出回答的优点和不足
- ✅ **自适应提问**：根据回答调整下一个问题
- ✅ **完整记录**：保存所有对话历史
- ✅ **评估报告**：生成详细的能力评估

## 故障排除

### 问题：面试结束时卡住

**原因**：面试评分低于6.0分会自动触发学习规划器（Agent 5）

**解决**：演示脚本已自动禁用此功能，无需额外处理

### 问题：LLM 调用超时

**原因**：网络问题或 API 限流

**解决**：
1. 检查网络连接
2. 确认 API Key 配置正确
3. 稍后重试

### 问题：回答不匹配问题

**原因**：面试官问题是动态生成的，预设回答可能不匹配

**解决**：修改 `user_inputs` 中的回答内容，使其更通用

## 与前端集成

### API 接口

```javascript
// 1. 开始面试
POST /chat/stream
{
  "message": "模拟后端技术面试",
  "session_id": ""
}

// 2. 回答问题
POST /chat/stream
{
  "message": "你的回答...",
  "session_id": "session-id"
}

// 3. 结束面试
POST /chat/stream
{
  "message": "结束面试",
  "session_id": "session-id"
}
```

### SSE 事件流

前端通过 SSE 接收实时更新：

```javascript
const eventSource = new EventSource('/chat/stream');

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  
  if (data.type === 'status') {
    // 更新状态栏
    updateStatus(data.node, data.message);
  } else if (data.type === 'message') {
    // 追加消息内容
    appendMessage(data.chunk);
  } else if (data.type === 'done') {
    // 面试结束
    eventSource.close();
  }
};
```

## 注意事项

1. **API Key 配置**：确保 `.env` 文件中配置了正确的 LLM API Key
2. **网络连接**：需要稳定的网络连接访问 LLM 服务
3. **面试轮数**：默认最多 8 轮，可在 `InterviewerAgent.MAX_TURNS` 修改
4. **评分标准**：评分范围 0-10 分，低于 6.0 分会触发学习建议
